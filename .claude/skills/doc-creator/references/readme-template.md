# README Template

## Document Structure

```markdown
# Project Name

Brief description of the project (1-2 sentences).

## Features

- Feature 1
- Feature 2
- Feature 3

## Installation

### Prerequisites
- Node.js >= 18
- npm or yarn

### Steps
```bash
git clone https://github.com/user/project.git
cd project
npm install
```

## Usage

### Basic Usage
```bash
npm start
```

### Configuration
Describe configuration options.

## API Reference

Link to detailed API documentation or brief overview.

## Contributing

1. Fork the repository
2. Create feature branch (`git checkout -b feature/name`)
3. Commit changes (`git commit -m 'Add feature'`)
4. Push to branch (`git push origin feature/name`)
5. Open Pull Request

## License

MIT License - see LICENSE file for details.
```

## Section Guidelines

### Title and Description
- Clear, descriptive project name
- One-line description of what the project does

### Features
- List 3-5 key features
- Use action verbs

### Installation
- List all prerequisites with versions
- Provide copy-paste commands
- Include platform-specific instructions if needed

### Usage
- Start with the simplest example
- Add advanced usage sections as needed
- Include configuration options

### Contributing
- Keep it simple for open source
- Link to CONTRIBUTING.md for detailed guidelines

### License
- State the license type clearly
- Link to full license file
